100ProSim - Energy Simulation System

User Manual - Land Use Data Management

What This Web App Does

100ProSim is a simulation tool for planning how to reach 100% renewable energy by 2045.

This web app helps you test and understand the energy transition scenario in a structured way:

  • Baseline year (Status): Represents the 2023 data for renewable production and energy demand
  • Target year (Ziel): Represents the projected/planned 2045 scenario
  • Main objective: Check how 2045 demand can be fully covered by renewable energy (100% renewable supply)
  • Four demand sectors: Demand is managed and evaluated across the main sectors shown in the app
  • User control: You can edit land use and demand assumptions, run recalculations, apply balancing, and compare results
  • Decision support: Use Renewable, WS (365 Tage), Bilanz, and Cockpit views to validate whether the system is balanced
Background: This web application is inspired by the Excel-based 100ProSim workflow and brings the same planning logic into an interactive simulation environment.

Step 1: Accessing Land Use Data

Navigate to the Land Use Data page from the sidebar menu

Figure 1: Land Use Data page showing all land use records with hectare values

Features

On the Land Use page, you can:

  • View all land use entries: See codes, names, status and target hectares
  • Edit values: Click on any row to modify hectare values
  • Monitor status: Check current vs. target values
  • Track recent updates: At the bottom of the page, the Recent Changes panel shows your latest land use changes
  • Review outcomes: Check Renewable and Bilanz pages after updates to verify the latest results

Step 2: Editing and Saving Land Use Data

Click on any land use row to edit the target hectare values

Figure 2: Editing land use hectare values and saving changes

When editing land use data:

  • Click Edit: Click the edit button on any land use row to modify values
  • Enter New Values: Input the desired target hectare value
  • Save All Values: Click Save All Values to store all current land-use edits together
  • Automatic recalculation: After saving, the system recalculates the dependent renewable values and refreshes the 365-day annual electricity outputs in the background
  • Review updated results: After the save job finishes, check Renewable Energy, Szenario-Abgleich, and Jahresstrom to verify the updated results
Tip: Land use changes directly affect renewable energy calculations, the WS/365-day values, and the annual electricity overview. After saving, always verify the updated results in Renewable Energy, Szenario-Abgleich, Jahresstrom, and Bilanz.

Step 3: Viewing Renewable Energy Overview

Navigate to the Renewable Energy page to view comprehensive energy data

Figure 3: Renewable Energy page showing editable values and the Recalculate Renewables button

The Renewable Energy page lets you review and update renewable-energy-related values:

  • Edit values: Update the user-editable renewable values directly on this page where applicable.
  • Recalculate Renewables: After editing a value, click Recalculate Renewables to rebuild the renewable calculations.
  • Page refresh: This recalculation updates the Renewable Energy page itself and also refreshes dependent values that rely on renewable outputs.
  • Status and target comparison: Review the resulting values across the renewable categories after recalculation.
  • Preparation for later steps: Use the recalculated values before checking Szenario-Abgleich, Jahresstrom, Bilanz, or Cockpit.
Important: If you edit a renewable input manually, always click Recalculate Renewables afterwards so the page and its dependent values are updated consistently.

Step 4: Managing Energy Consumption (Verbrauch)

Navigate to the Verbrauch page to adjust energy consumption percentages

Figure 4: Verbrauch page showing adjustable energy consumption percentages by sector

The Verbrauch (Consumption) page allows you to manage energy distribution:

  • Adjust Percentages: Click on any sector to modify its energy consumption percentage share
  • Automatic Rebalancing: When you increase one sector's percentage, other sectors automatically adjust (decrease) to maintain a total of 100%
  • Sector Distribution: View and modify energy allocation across different consumption sectors (Industry, Transport, Households, etc.)
  • Real-time Updates: Changes are immediately reflected in the energy balance calculations
  • Interactive Controls: Use the arrow indicators to easily identify where adjustments can be made
  • Save at the bottom: After editing values, always use the Save button at the bottom of the page
Important: The system ensures all sector percentages always sum to 100%. When you change one percentage, the system automatically redistributes the remaining percentage across other sectors proportionally. If the update does not appear immediately after one click, press the Save button once more.

Step 5: Recalculate Renewables After Verbrauch Changes

After changing anything in Verbrauch, go to Erneuerbare Energien and click "Recalculate Renewables"

Figure 5: Renewable Energy page with the "Recalculate Renewables" button

This recalculation step is required after Verbrauch updates:

  • Sync formulas: Rebuilds renewable status and target values using the latest inputs
  • Keep data current: Ensures the Renewable page reflects all recent Verbrauch changes
  • Prepare next steps: Use this before checking Cockpit, WS (365 Tage), or Bilanz results

Step 6: Check Bilanz Balance Status

Open "Bilanz" and check the WS-365 Balance Status chart

Figure 6: Bilanz page showing WS-365 balance status

Figure 6b: Bilanz page showing whether each sector is balanced

This step gives a quick visual check of WS-365 balancing:

  • If it looks balanced, it is balanced: When the status and drift indicators show balanced values, the 365-day cycle is closed.
  • Sector check: Use the sector-wise Bilanz view to confirm each sector is balanced.
  • Visual confirmation: Use this chart as the fast final check after balancing.

Step 7: Szenario-Abgleich - Balance and Comparison Page

Use Szenario-Abgleich to review the current storage balance, compare the solar and wind balancing options, and apply the required balance actions.

Figure 7a: Szenario-Abgleich page showing the solar balance path

Figure 7b: Szenario-Abgleich page showing the wind balance path

What Szenario-Abgleich does: This page shows the current comparison of solar and wind balancing options based on the latest model state. To apply a balance safely, always use the two buttons in the correct order.

  • Solar order: First click WS Balance Solar. Only after that finishes, click Sector + WS Solar Balance.
  • Wind order: First click WS Balance Wind. Only after that finishes, click Sector + WS Wind Balance.
  • Why this order matters: The first WS button performs the WS balancing step. The second button then applies the combined sector plus WS balance on top of that updated state.
  • Goal Seek ausführen: Rebuilds the comparison basis and updates the preview values shown in the solar and wind balance cards.
  • Aktualisieren: Refreshes the displayed comparison values from the current database state.
  • Final check: After the correct two-button sequence, confirm the final state in Bilanz and Jahresstrom.

Step 8: Jahresstrom / Annual Electricity

After balancing, use Jahresstrom to review both the annual electricity flow diagram and the detailed 365-day table in one page.

Figure 8a: Annual electricity flow diagram showing yearly electricity generation, storage, reconversion, curtailment, and end-use flow

The first part of Jahresstrom presents the annual electricity flow diagram and helps you validate the system-level yearly picture:

  • Annual generation picture: Shows the yearly electricity contributions from bio, PV, wind, and hydro/geothermal.
  • Curtailment and storage path: Shows how surplus electricity is curtailed, routed into electrolysis, stored, and later reconverted.
  • Storage indicators: Displays storage capacity, conversion efficiencies, and storage efficiency values derived from the current 365-day result.
  • Final electricity delivery: The final grid block shows the annual electricity delivered to end consumption after storage and reconversion are considered.
  • Scenario title: The page header reflects the currently active scenario/version and its date when a saved scenario has been restored.

Figure 8b: Jahresstrom page showing the detailed 365-day table that was moved from Szenario-Abgleich

The second part of Jahresstrom contains the detailed 365-day dataset:

  • 365-day analysis: Displays the day-by-day values used for the yearly electricity simulation.
  • Detailed verification: Lets you inspect the columns for surplus, storage, curtailment, reconversion, and charge-level behavior across the whole year.
  • Traceability: Helps confirm how the yearly diagram values are built from the underlying 365-day calculation results.
  • Reporting support: Use this part of the page when you need a detailed check of the annual system behavior after balancing.

Step 9: Understanding Visual Representations

Interpret the Cockpit sector chart to verify whether renewable production is aligned with demand

Figure 9a: Cockpit sector view showing sector deficits with red delta markers

Figure 9a represents the imbalance case and highlights required action:

  • Paired sector bars: Blue shows sector demand (Verbrauch), green shows renewable supply (Erneuerbare) for the same sector.
  • Red delta badges (e.g., Δ -404): Demand is higher than renewable production in that sector.
  • Sector-level gap detection: Each red marker quantifies the deficit directly in MWh/a, enabling targeted correction.
  • Follow-up action: Use WS/Bilanz balancing functions to close these sector gaps, then re-check the cockpit for updated markers.
  • Validation loop: The chart serves as a visual control step after each recalculation and balancing cycle.

Figure 9b: Cockpit sector view with balanced demand and renewable production

Figure 9b represents the balanced case and should be read as follows:

  • Balanced indicator: The light-blue Perfekt badge means the sector difference is within the accepted tolerance and considered balanced.
  • Operational meaning: In this view, balancing actions have aligned renewable output with sector demand at cockpit level.
  • Quick decision support: You can immediately confirm that no sector requires corrective balancing.
  • Stable visual state: Similar bar heights with Perfekt markers indicate a robust and consistent system state.
Tip: Use Step 9 as a fast visual quality check: Perfekt badges indicate balanced sectors, while red delta badges indicate where additional balancing is still required.

Step 10: Save Scenario Snapshots and Continue Later

At any point, you can save the current situation as a scenario snapshot and return to it anytime.

Figure 10: Scenario snapshot view for saving and reopening simulation states

Use scenario snapshots to keep your work organized and easy to revisit:

  • Save current state: Capture the current values and balance state whenever you reach an important point.
  • Return anytime: Open the saved scenario later and continue from the same situation without starting over.
  • Compare alternatives: Store different planning variants (for example, different demand or land-use choices) and compare outcomes.
  • Work safely: Snapshotting before major changes gives you a clean fallback point.

Step 11: Create, Save, Restore, Rename, and Delete Scenarios

Use the Scenarios menu in the top navigation to save named scenario versions and restore them later.

Figure 11: Scenario menu showing create, restore, rename, and delete actions for saved versions

The current web application supports structured scenario version management:

  • Save Current Scenario: Stores the complete current model state under a user-defined scenario name.
  • Restore: Loads a previously saved scenario version back into the current session.
  • Rename: Lets you update the scenario name if the version should be described more clearly.
  • Delete: Removes outdated or no-longer-needed scenario versions.
  • Practical use: Save a scenario before major changes, test another variant, and restore the earlier version if needed.
  • Link to Jahresstrom: When a scenario is restored, the annual electricity page header reflects the active scenario name and timestamp.
